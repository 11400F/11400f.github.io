P8180 Unmemorable题解

题目描述

Unputdownable 手中有一个长度为 $n$ 的排列 $a$。

他在练习单调栈的时候用程序对于每一个 $i$ 求出了最大的 $li$ 使得 $a{li} < a_i$ 且 $l_i<i$，以及最小的 $r_i$ 使得 $a{r_i}i$。

特别的，若这样的 $l_i$ 不存在，则定义为 $0$，不存在的 $r_i$ 则定义为 $n+1$。

某日 Unputdownable 忘记了排列 $a$，而且只剩余分别重排后的 $l$ 和 $r$ 数组了，你能帮助他还原原来的排列 $a$ 吗？

随后由于他发现无法还原 $a$，你只要告诉他有多少种可能的原排列 $a$。

答案对于 $998244353$ 取模，数据保证至少存在一种方案。

输入格式

第一行输入一个整数 $n$。

第二行输入 $n$ 个整数，表示重排后的 $l$。

第三行输入 $n$ 个整数，表示重排后的 $r$。

输出格式

输出一行，包含一个整数，表示可能的排列数量对 $998244353$ 取模后的值。

SAMPLE I/O

I

O

数据范围

对于 $100\%$ 的数据，$1 \leq n \leq 10^6$，$0 \leq l_i,r_i \leq n+1$，数据保证至少存在一种方案。

思路

转换为正常的 l 序列和 r 序列

这里题目提供了乱序的 $l$ 序列和 $r$ 序列，我们看他很不顺眼，想把它转换成正常的 $l$ 和 $r$ 序列。怎么转换呢？

想想 $l$ 和 $r$ 序列的定义。${l_i}$ 指的是 $i$ 左边离 $i$ 最近的小于 ${p_i}$ 的数。

回顾正常构建 $l$ 序列的步骤（$O(n)$）：

我们现在有一个栈和一个排列 $p$，从左往右扫序列 $p$，扫到第 $i$ 个数，就检查一下栈顶：若栈顶的数 $s$ 比 $p_i$ 小，那么就说明 $p_i$ 找到第一个比它小的数了。把 $p_i$ 插入栈。否则说明比 $p_i$ 小的数还在栈的下面。

同时，我们也发现了$s$ 的 $r$ 也找到了，就是 $i$。因为这时 $p_i$ 比栈顶的数小。而 $s$ 的 $l$ 在之前确定了（因为 $l_s$ 不可能跑到 $s$ 的右边去），所以 $s$ 的 $l$ 和 $r$ 都找到了，就把 $s$ pop 出栈，以便 $p_i$ 和后面的数确定 $l$ 值。


正常的求解L和R序列的图解

于是我们可以发现，一个数的位置 $i$ 前面有多少个比它大的数，那么 $r$ 序列里面就会有多少个 $i$。相应的， $i$ 后面又多少个比他大的数，那么 $l$ 序列里面就会有多少个 $i$。我们可以根据这个性质来重新建立起正确的 $l$ 和 $r$ 序列。

这里不用管他的数值，只用管他的下标（就是上图里的紫色数值）。记录 $l$ 和 $r$ 序列里的每个数出现的次数 $cntl_i$ 和 $cntr_i$，然后根据出现次数倒推。

例如：假设现在扫到了 $i=3$， $cntl_i = 2$，那么就先往栈里删除两个数，记录这两个数的 $r$ 值为 $i$，然后再把 $i$ 压入栈。

将序列转换为 DAG（有向无环图）

我们回到定义，$l$ 序列和 $r$ 序列的意义是什么？是分别在左边（$l$）和右边（$r$）第一个比他小的数。那就说明，我们构建出的序列必须要满足 $p{l_i} < p_i$，和 $p{r_i} < p_i$ 的关系。

这种限制可以转化成一条有向边，有 $p{a} < p{b}$ 的限制就只需要从 $a$ 向 $b$ 连一条有向边即可。于是我们就得到了一个有向无环图。为什么无环？因为不存在 $a<b, b<c$ 且 $c<a$ 的关系。

建成了一个有向无环图之后，为了满足这种关系，可以从最小的数，就是入度为 $0$ 的点开始赋最小值，然后依次往里扩散，赋的值也相应的比之前大。发现这种方法是不是非常像求拓扑序？于是这道题的目标就变成了：给一个 DAG，求他的拓扑序计数。

转图为树

对于普通图的拓扑序计数，他的复杂度是很大的，放到这道题是不现实的。我们再回到图上，发现原来的图他的边太多而且太复杂了！$n = 10^6$，用一般的图或者 DAG 拓扑序计数非常容易炸。想想怎么简化这张图。

我们发现，对于一个点 $i$ 的 $li$ 到 $r_i$ 的区间，除了下标为 $l_i$ 和 $r_i$ 的点，其他的数都是大于等于 $p_i$ 的。并且 $p{li}$ 和 $p{r_i}$ 都是小于 $p_i$ 的，于是 $l_i$ 和 $r_i$ 双方都很有可能成为对方的 $l$ 的 $r$。

只有两种情况：

  • 若$p{l_i} < p{ri}$，也是 $l{r_i} = l_i$，那么就说明 $l_i$ 是 $r_i$ 左边第一个小于点 $r_i$ 的点，于是 $l_i$ 肯定会向 $r_i$ 连边。那么就只需 $r$ 向 $i$ 连边。
  • 否则，$p{l_i} > p{ri}$，也是 $r{l_i} = r_i$，那么就说明 $r_i$ 是 $l_i$ 右边第一个小于点 $l_i$ 的点，于是 $r_i$ 肯定会向 $l_i$ 连边。那么就只需 $l$ 向 $i$ 连边。

但是，要特判：如果 $l_i = 0$，就只能 $r$ 向 $i$ 连边；如果 $r_i = n+1$，就只能 $l$ 向 $i$ 连边。

于是，从每一个点都伸出来了一条边，这样就有 $n$ 条边，但是最小值的 $l_i = 0, r_i = n-1$，就相当于是向虚空连边，真实存在的边只有 $n-1$ 条。$n$ 个点，$n-1$ 条边，每条边都有度数……

它是一棵树！

我们已知一棵大小为 $siz$ 的树，如何求它的拓扑序计数？

现在先不管合不合法，把每个拓扑序分配给每一个点，那么一共就有 $siz!$ 种情况。其中只考虑这个根，让根满足条件的就是根节点的拓扑序小于其他所有节点，就是 $1$。

枚举这 $siz!$ 种情况，其中 $1$ 只能在第一个位置。其他的位置都不合法。那么这个概率就是 $\frac{1}{siz}$。然后再递归下去，求出每个节点的合法概率。把这些点的概率相乘，就变成了总的合法概率。再乘上总共情况 $siz!$，即为总共合法情况数：

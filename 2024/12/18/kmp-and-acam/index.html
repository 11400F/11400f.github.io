KMP 和 AC 自动机

 

你现在有两个字符串 $s$ 和 $t$，长度分别为 $n$ 和 $m$。你现在要求 $t$ 串作为 $s$ 串的子串时，出现的位置。

暴力匹配是非常容易给你卡成复杂度 $O(nm)$ 的，所以我们要用一个非常厉害但难想的算法：KMP算法。

KMP

暴力匹配的算法因为在某一个地方失配了之后，要回溯到开头的下一位，就浪费了很多时间。而 KMP 算法就利用了字符串的最长公共前后缀，完美地避免了在主串 $s$ 的回溯问题。

于是我们就可以考虑，如果在某一个位置失配了，就提取之前匹配的部分，找到它的最长公共前后缀（就是一个字符串最长的相同的前、后缀，例如串 $abcabc$ 的最长公共前后缀就是 $abc$，串 $aafaafaa$ 的最长公共前后缀就是 $aafaa$，串 $aaaaa$ 的最长公共前后缀就是 $aaaa$），因为这两个部分是相同的，所以可以直接从这一位开始匹配。

例子：

文本串 $s$：$maimaidxmaimaicba$

模式串 $t_1$：$maid$，$t_2$：$maimaic$

对于 $t_1$：

  • 第一次匹配：在 $i=4, j=4$ 处失配，因为 $mai$ 串无公共前后缀，所以让 $i=4, j=1$。
  • 第二次匹配：在 $i=7, j=4$ 处匹配，则结束。

对于 $t_2$：

  • 第一次匹配：在 $i=7, j=7$ 处失配，因为之前成功匹配的 $maimai$ 串有最长公共前后缀 $mai$，则让 $i=7, j=4$，继续匹配。
  • 第二次匹配：在 $i=7, j=4$ 处失配。$d \neq c$。而 $mai$ 串无公共前后缀，则让 $i=8, j=1$。
  • 第三次匹配：在 $i=8, j=1$ 处失配。因为空串无公共前后缀，则让 $i=9, j=1$。
  • 第四次匹配：在 $i=15, j=7$ 处匹配。结束。

如上述例子，我们可以利用之前成功匹配的串的最长公共前后缀，让 $i$ 指针避免回溯。这样我们的时间复杂度就可以控制在 $O(n+m)$。

那么我们该如何获取对于模式串的每一个 $j \in [1, m]$，串 $t$ 的 $[1,j]$ 部分的最长公共前后缀呢？

我们先把串 $t$ 的 $[1,j]$ 部分的最长公共前后缀记为 $nxtj$。假设我们知道了对于 $j$ 的 $nxt$ 值，如何求 $nxt{j+1}$？

  • 如果 $t{j+1} = t{nxtj + 1}$，那么 $nxt{j+1} = nxt_{j} +1$。

  • 如果 $t{j+1} \neq t{nxt_{j} +1}$，则 $j ← nxt_j$，继续判断，直到匹配成功之后 $+1$ 或是匹配失败到 $j=0$ 为止。

有没有发现，这个思路和 KMP 求 $t$ 串在 $s$ 串中的匹配很像？思路非常相似。

AC 自动机

AC 自动机和 KMP 类似，不过 AC 自动机是实现多模式串和一个文本串的匹配的。

那么我们在遍历某节点的 $26$ 个儿子的时候，若该儿子存在，那么就以它的儿子为起点连接儿子的回跳边。

那如果某个儿子不存在呢？

我们来介绍另外一种边：转移边。某个节点的转移边则指向当前节点的回跳边所指向的儿子。它少了个什么？少了一个爹。

注意：这里把转移边定义为了 $ch{i, j}$，是因为转移边就是 $i$ 对于字母 $j$ 的转移边所指的节点 $ch{i, j}$ 的最短路。如果有真实的从 $i$ 连向 $j$ 的边，那么就不要连转移边。如果我们想从点 $i$ 出发走边权为 $j$ 的边的话，我们完全没有必要先跳回跳边再走边权为 $j$ 的边。直接建一条转移边就可以从 $i$ 走边权为 $j$ 的边了。

类似的问题：

有 $N$ 个由小写字母组成的模式串以及一个文本串 $T$。每个模式串可能会在文本串中出现多次。你需要找出哪些模式串在文本串 $T$ 中出现的次数最多。

这种问题与上面的问题思路差不多，只需要插入时将最后的位置 $p$ 在 $cnt$ 上对应的值为插入的串的编号，最后统计时只要这个地方有编号就将此地答案 $+1$，最后取最大值即可。

应用：ACAM + 一般DP

P3041

Bessie 在玩一款游戏，该游戏只有三个技能键 A，B，C 可用，但这些键可用形成 $n$ 种特定的组合技。第 $i$ 个组合技用一个字符串 $s_i$ 表示。

Bessie 会输入一个长度为 $k$ 的字符串 $t$，而一个组合技每在 $t$ 中出现一次，Bessie 就会获得一分。$s_i$ 在 $t$ 中出现一次指的是 $s_i$ 是 $t$ 从某个位置起的连续子串。如果 $s_i$ 从 $t$ 的多个位置起都是连续子串，那么算作 $s_i$ 出现了多次。

若 Bessie 输入了恰好 $k$ 个字符，则她最多能获得多少分？

这道题可以先把这些串插进一个 ACAM 里面，然后给是这个字符串结尾的节点的 $cnt$ 加上 $1$。

然后我们可以设 DP 状态：$dp_{i, j}$ 表示现在 Bessie 输入了第 $1\sim i$ 个字符，她现在输入的字符导致指针走到了 ACAM 的 $j$ 号节点所能取得的最多的分数。

然后我们考虑转移：如果存在一条边 ${ch{j, alp}}$，那么就可以转移：$dp{i+1, {ch{j, alp}}} = \max(dp{i, j} + {cnt{ch{j, alp}}})$。

并且注意：最开始要 memset dp 数组为负无穷，然后设置 $dp_{0, 0} = 0$。

最后取 $\displaystyle\max{0\le j\le idx}(dp{k, j})$ 即可。

这道题因为 $n$ 只有 $1e5$，于是就可以考虑枚举 $i$，算每一个 $i$ 对答案的贡献。这样我们就要要求算后面那个 $\Sigma$ 的时候时间复杂度要尽量小。

因为这些数他最大也只有 $5$ 位，我们就可以在枚举 $i$ 的时候，把 $i$ 的所有子串全部插进一个 ACAM 里面。我们容易发现，当你拿一个文本串去和 ACAM 里的模式串匹配的时候，这两个串的 $\text{LCS}$ 就是指针走过的节点的深度的最大值。然后求 $j$ 的过程可以使用数位 DP 来做。

具体地说，我们可以再枚举 $i$ 和 $j$ 的 $\text{LCS} = k$，因为最多也只到 $5$，所以无伤大雅。对于每一个 $\text{LCS} = k$，分别数位 DP 求：和 $i$ 的 $\text{LCS}$ 长度为 $k$ 的小于 $n$ 的数 $j$ 到底有多少个。设 $dp_{bit, p, nowl, is0 = (0/1), islim = (0/1)}$ 表示：现在枚举到了从低到高的第 $bit$ 位，现在在 ACAM 的编号为 $p$ 的点上，当前的最大 $\text{LCS}$ 为 $nowl$，是否含有前导 $0$，是否以抵达枚举的上界 $n$（这两项是数位 DP 的基本操作）。

然后终止条件就是当 $bit = 0$ 的时候，判断一下 $nowl$ 是否等于当前枚举的 $\text{LCS}$：$k$。如果等于返回 $1$，否则返回 $0$。然后其他的统计就是数位 DP 的基本操作。记得最后给答案算贡献时要乘上 $k$，否则会出错！

P2444

二进制病毒审查委员会最近发现了如下的规律：某些确定的二进制串是病毒的代码。如果某段代码中不存在任何一段病毒代码，那么我们就称这段代码是安全的。现在委员会已经找出了所有的病毒代码段，试问，是否存在一个无限长的安全的二进制代码。

示例：

例如如果 ${011, 11, 00000}$ 为病毒代码段，那么一个可能的无限长安全代码就是 $010101 \ldots$。如果 ${01, 11, 000000}$ 为病毒代码段，那么就不存在一个无限长的安全代码。

现在给出所有的病毒代码段，判断是否存在无限长的安全代码。

这道题可以先造出这些串的 ACAM，然后观察它的性质：我们设 ACAM 上的某个表示的字符串存在病毒代码的点为病毒点，那么存在无限长的安全代码当且仅当这个 ACAM 上存在一个不经过病毒点的环。

于是我们可以在造 ACAM 的同时预处理出哪些点是病毒点，然后从根开始跑一遍 DFS 查找有无符合上述条件的环即可。

具体地说，我们定义一个状态数组 $stat$ 表示每个点的状态：若值为 $0$，表示这个点没有被 DFS 到过；若值为 $1$，表示现在正在 DFS 这个点或从这个点连出去的点；若值为 $2$，则表示这个点已经被遍历过了。根据定义，就要刚刚开始遍历这个点的时候设置这个点的 $stat$ 值为 $1$，把这个点连出去的点遍历完了之后就设这个点的 $stat$ 为 $2$。

然后 DFS 时，先判断终点是否为病毒点。如果是则 continue。若不是，则判断其状态。若其为 $0$，则继续深搜。若其为 $1$，那么就可以设置答案为 true 了。DFS 完了之后即可输出。

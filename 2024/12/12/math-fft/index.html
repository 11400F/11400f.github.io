多项式入门：快速傅里叶变换

前传：一点点计算几何：最小圆覆盖

我们现在有两个多项式： \[ A(x) = a_0 + a_1x + a_2x^2 + a_3x^3 + \cdots + a_{n-1}x^{n-1}\\ B(x) = b_0 + b_1x + b_2x^2 + b_3x^3 + \cdots + b_{m-1}x^{m-1} \] 我们想算他们两个乘起来的多项式 \(C(x)\) 的每一项的系数，该怎么算呢？

法一：系数乘法

直接暴力算，\(c_x = \sum_{i=0}^{x} a_ib_{x-i}\)。我们成功地得到了答案，但是这样子做的话时间复杂度是 \(O(n^2)\)，你这 \(n\) 一大，程序就跑不动了。但是我们还有其他方法：

法二：点值乘法

众所周知，两点可以确定一条直线，三个点可以确定一条抛物线，那么以此类推，\(n\) 个不同的点就可以确定 \(n-1\) 次函数的图像了！

于是我们选 \(n+m-1\) 个不同的 \(x\) 值 \(x_0, x_1, x_2, \cdots, x_{n+m-2}\) 进去，然后算出 \(A(x_0), A(x_1), \cdots, A(n+m-2)\) 和 \(B(x_0), B(x_1), \cdots ,B(n+m-2)\)，因为 \(C(x) = A(x) \times B(x)\)，所以可以直接 \(O(n)\) 算出 \(C(x_0), C(x_1), \cdots, C(x_{n+m-2})\)，最后用插值法计算 \(C\) 的系数即可。时间复杂度依然为 \(O(n^2)\)。

那么我们该如何优化呢？法一似乎没有多大前途，我们来看法二该怎么优化。这就要请出我们今天的主角：FFT！

哦不，不是这个FFT

快速傅里叶变换

快速傅里叶变换可以把法二的求点值（即 \(A(x_k)\)、\(B(x_k)\) 的值）和最终求系数的过程的时间复杂度都优化到 \(O(n\log n)\)。不过需要借助复数运算。

那么我们首先回顾以下复数有关的知识吧！

复数

众所周知，复数的基本形式是 \(z=x+iy\)。\(x\) 是实部，\(y\) 是虚部。

加法和减法就是实部和虚部分别相加或相减。而乘法就是运用和乘法分配律相似的思路：

\(z_1 \times z_2 = (x_1x_2-y_1y_2) + i(x_1y_2 + x_2y_1)\)。

单位圆、单位根

我们可以把复数 \(x+iy\) 看作一个点 \((x,y)\) 把他搬到复平面上，复平面就是 \(x\) 轴表示实部，\(y\) 轴表示虚部的一个平面。

而单位圆就是一个圆心在 \((0,0)\) 并且半径为 \(1\) 的圆。

假设圆上的某点 \(z\) 绕点 \(O\) 顺时针旋转 \(\theta\) 可以落在点 \((1,0)\) 上，那么这个点 \(z = \cos\theta + i\sin\theta\)。

证明是好证的，可以过该点做一条向 \(x\) 轴的垂线，然后用三角函数求即可。

我们把这个圆 \(n\) 等分，那么这 \(n\) 个点 \(z_0, z_1, \cdots, z_{n-1}\) 就都是方程 \(z^n = 1\) 的 \(n\) 个复数根。

那么我们定义：单位根 \(\omega_{n}^{k}\) 是把这个圆 \(n\) 等分，然后从 \((1,0)\) 开始逆时针数的第 \(k\) 个点，也就是在这 \(n\) 个复数根里第 \(k\) 个根（这里的 \(k\) 是从 \(0\) 开始数的。其实 \(\omega_n^n = \omega_n^0\)）。

我们知道，把一个圆分为了 \(n\) 份，那每一份的角度就是 \(\frac{2\pi}{n}\)，第 \(k\) 份的 \(\theta\) 其实就是 \(\frac{2\pi k}{n}\)。那么就可以得出：

单位根 \(\omega_n^k = \cos\frac{2\pi k}{n} + i\sin\frac{2\pi k}{n}\)。

单位根有一些性质：

  • 相乘时：\(\omega_{n}^{k}\omega_{n}^{m} = \omega_{n}^{k+m}\)
  • 乘方时：\((\omega_{n}^{k})^{m} = \omega_{n}^{km}\)（其实就是 \(m\) 个 \(\omega_{n}^{k}\) 相乘）

而 FFT 在运算的时候要求 \(n = 2^j(j\in ℕ)\)，也就是 \(n\) 必须是 \(2\) 的整数次幂，那么它也就有了以下几种特殊性质：

  1. 周期性：\(\omega_{n}^{k+n} = \omega_{n}^{k}\)。也就是它其实是 \(n\) 个一循环的。
  2. 对称性：\(\omega_{n}^{k+n/2} = -\omega_{n}^{k}\)。因为这里的 \(n\) 它是 \(2\) 的整数次幂，所以可以证明这一点。就如上图的 \(\omega_8^1\) 和 \(\omega_{8}^{5}\)。
  3. 折半性：\(\omega_{n}^{2k} = \omega_{n/2}^{k}\)。也比如上面的 \(\omega_8^2\) 和 \(\omega_{4}^{1}\)。

快速傅里叶变换

于是我们就可以开始 FFT 来求一个多项式的点值了！

我们现在有 \(A(x) = a_0 + a_1x + a_2x^2 + \cdots a_{n-1}x^{n-1}\)，我们要求它在 \(n\) 个不同的值时的点值。

我们可以利用上面的单位根，把这 \(n\) 个不同的 \(x\) 分别赋为 \(\omega_{n}^{0}, \omega_{n}^{1}, \cdots, \omega_{n}^{n-1}\)。然后来求点值。

于是我们考虑把 \(A(x)\) 拆开： \[ A(x) = (a_0 + a_2x^2 + a_4x^4 + \cdots + a_{n-2}x^{n-2}) + x \times(a_1 + a_3x^2 + a_5x^4 + \cdots + a_{n-1}x^{n-1}) \] 然后我们再设 \(A_1(x) = a_0 + a_2x + a_4x^2 + \cdots + a_{n-2}x^{n/2-1}\)，\(A_2(x) = a_1+a_3x + a_5x^2 + \cdots + a_{n-1}^{n/2-1}\)。

这时我们只要把 \(x^2\) 代入 \(A_1\) 和 \(A_2\) 函数就会发现： \[ A(x) = A_1(x^2) + xA_2(x^2) \] 然后我们将 \(\omega_{n}^{k} (k < n/2)\) 代入 \(x\) 得： \[ \begin{aligned} A(\omega_{n}^{k}) &= A_1((\omega_n^k)^2) + \omega_n^kA_2((\omega_{n}^{k})^2)\\ &= A_1(\omega_{n}^{2k}) + \omega_{n}^{k}A_2(\omega_{n}^{2k}) \\ &= A_1(\omega_{n/2}^{k}) + \omega_{n}^{k}A_2(\omega_{n/2}^{k}) \end{aligned} \] 于是我们发现当 \(k < n/2\) 时，我们可以将其拆成两半算。

我们再将 \(\omega_{n}^{k+n/2}\) 代入 \(x\) 得： \[ \begin{aligned} A(\omega_{n}^{k+n/2}) &= A_1((\omega_{n}^{k+n/2})^2) + \omega_{n}^{k+n/2}A_2((\omega_{n}^{k+n/2})^2) \\ &= A_1(\omega_{n}^{2k+n}) - \omega_{n}^{k}A_2(\omega_{n}^{2k+n})\\ &= A_1(\omega_{n}^{2k}) - \omega_{n}^{k}A_2(\omega_{n}^{2k})\\ &= A_1(\omega_{n/2}^{k}) - \omega_{n}^{k}A_2(\omega_{n/2}^{k}) \end{aligned} \] 发现和上面有什么不同的没有？其实答案就只是变了一个符号而已。

所以对于所有的 \(0 \le k < n\)，我们都可以将 \(A(\omega_{n}^{k})\) 拆为两半来算。

只要我们算出了 \(A_1\) 和 \(A_2\) 的点值，我们就可以根据这个来推出 \(A\) 的点值。

于是我们就可以利用分治的思想，将 \(a_i\) 里 \(i\) 为偶数的部分塞给 \(A_1\)，将 \(i\) 为奇数的部分塞给 \(A_2\)。然后递归下去。

求出 \(A_1\) 和 \(A_2\) 的点值之后，枚举 \(0\le k < n/2\)，将 \(A(\omega_{n}^{k})\) 赋为 \(A_1(\omega_{n/2}^{k}) + \omega_{n}^{k}A_2(\omega_{n/2}^{k})\)，将 \(A(\omega_{n}^{k+n/2})\) 赋为 \(A_1(\omega_{n/2}^{k}) - \omega_{n}^{k}A_2(\omega_{n/2}^{k})\)，然后再递归上去即可。

边界条件就是：当 \(n = 1\) 时，直接返回该数组即可。因为其实 \(\omega_{1}^{1}\) 就等于 \(1\)。

最后，我们发现，传进去的是参数数组，返回的就是点值数组了。这就是 FFT 的神奇之处！

现在，我们用快速傅里叶变换求出了多项式 \(A\) 和 \(B\) 的点值，然后将他们乘起来得到了多项式 \(C\) 的点值。那么我们现在该怎么用 \(O(n\log n)\) 的时间复杂度来根据这个来倒推回 \(C\) 的参数呢？

就要用快速傅里叶逆变换！

快速傅里叶逆变换

假设多项式 \(A(x) = a_0 + a_1x + a_2x^2 + a_3x^3 + \cdots + a_{n-1}x^{n-1}\)，代入了 \(\omega_n^0,\omega_n^1,\omega_n^2,\cdots,\omega_n^{n-1}\) 求出来的点值为 \(y_0,y_1,y_2,\cdots,y_{n-1}\)，我们要还原出多项式 \(A\) 的系数。

我们知道，\(y_k = \displaystyle\sum_{i=0}^{n-1}a_i(\omega_{n}^{k})^i\)。于是我们可以构造多项式 \(B(x) = y_0 + y_1x + y_2x^2 + \cdots + y_{n-1}x^{n-1}\)。

然后把这 \(n\) 个单位根的倒数 \(\omega_{n}^{0}, \omega_{n}^{-1}, \omega_{n}^{-2}, \cdots, \omega_{n}^{-(n-1)}\) 代入 \(B(x)\) 得到的点值设为 \(z_0, z_1, z_2, \cdots, z_{n-1}\)。

注：这里可以有倒数，因为 \(\omega_{n}^{k} = \cos\theta + i\sin\theta\)，那么 \(\frac{1}{\omega_{n}^{k}} = \frac{1}{\cos\theta + i\sin\theta} = \cos\theta - i\sin\theta = \cos(-\theta) + i\sin(-\theta) = \omega_{n}^{-k}\)。

然后 $$ \[\begin{aligned} z_k &= \displaystyle\sum_{i=0}^{n-1}y_i(\omega_{n}^{-k})^i\\ &= \sum_{i=0}^{n-1}\sum_{j=0}^{n-1} a_j(\omega_{n}^{i})^j\cdot(\omega_{n}^{-k})^i\\ &= \sum_{i=0}^{n-1}a_j\sum_{j=0}^{n-1}(\omega_{n}^{j-k})^i \end{aligned}\]

$$ 我们记后面的那一项和式 \(\sum_{j=0}^{n-1}(\omega_{n}^{j-k})^i\) 为 \(S\)。

那么这个 \(S\) 分两种情况：

  • \(j = k\)，此时 \(\omega_{n}^{0} = 1 + 0i\)。于是 \(S = n\)。
  • \(j \neq k\)，此时根据等比数列公式可得：\(S = \frac{(\omega_{n}^{j-k})^n-1}{\omega_{n}^{j-k}-1} = \frac{(\omega_{n}^{n})^{j-k}-1}{\omega_{n}^{j-k}-1}\)。因为 \(\omega_{n}^{n} = 1+0i\)，所以原式答案为 \(0\)。

所以 \(z_k = na_k\)，即 \(a_k = \frac{z_k}{n}\)。

我们发现这两个代码：变换和逆变换，只有一个地方，就是第九行的 sin(2*Pi/n) 的符号不同，于是我们可以加一个变量 op，正变换时赋为 \(1\)，逆变换时赋为 \(-1\)，然后那个地方就改为 op*sin(2*Pi/n) 即可。

并且注意：如果题目给了我们最高 \(n\) 次的 \(A\) 多项式和最高 \(m\) 次的 \(B\) 多项式，那么 \(C\) 多项式最高 \(n+m\) 次，因为至少 \(n+m+1\) 个点才能确定 \(n+m\) 次函数，于是我们扔进 FFT 函数里的那项数组长度必须是大于 \(n+m\) 的第一个 \(2\) 的整数次方的数。

FFT 优化

我们现在可以给他优化。

我们首先来画一个示意图，表示现在递归法的过程：

我们来看原序列（最上层）顺序和返回序列（最下层）顺序的关系：

原序列 0 1 2 3 4 5 6 7
二进制 000 001 010 011 100 101 110 111
返回序列 0 4 2 6 1 5 3 7
二进制 000 100 010 110 001 101 011 111

看出来了什么没有？原序列每个数的二进制把他颠倒过来就是返回序列每个数的二进制！e.g. \((3)_{10} = (011)_2 \stackrel{swap}{\longrightarrow} (110)_2 = (6)_{10}\)。

那么我们发现，新序列的每一个元素下标就是原序列的元素下标的二进制翻转，这个变换叫做位逆序变换。

我们可以先让原序列位逆序变换，得到最底下的序列，然后再自下而上合并就行了。

于是现在我们就需要求 \([0,n-1]\) 中每一个数它的二进制数颠倒过来是什么数。

我们有一个数组 \(R\)，来存每一个数它在 \(k = \log_2 n\) 二进制位下的二进制翻转的数。假设我们先把 \(R_{1} \sim R_{i-1}\) 都求出来了，我们现在要求 \(R_i\)。

我们可以这么做：先把 \(R_{i/2}\) 提出来，然后让它除以 \(2\)，最后在最高位填上 \(i\) 的个位二进制位即可。

举个例子：我们要求 \(R_{(11001)_2}\)，那么就可以先求得 \(R_{(01100)_2} = (00110)_2\)（前四位），然后将它除以 \(2\) 变为 \((00011)_2\)，最后，因为个位为 \(1\)，那么就在最高位上填 \(1\)，答案就是 \((10011)_2\)。

这种方法的原理其实就是先翻转前 \(k-1\) 位，然后最后再把原数的最后一位提到新数的最高位去，就完成了一次翻转的过程。

我们可以用一个公式概括：\(R_i = \lfloor\frac{R_{i/2}}{2}\rfloor + [i为奇数] \times (\frac{n}{2})\)。

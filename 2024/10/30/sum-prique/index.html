（二叉）堆算法总结

11490DX Lv4
  2024-10-30 20:05:31 2024-10-30 20:05:31 Created   2024-10-31 18:34:30 2024-10-31 18:34:30 Updated  
  • C++ 
  • | 数据结构 
 

 

堆是一种数据结构，经常被用来维护一组数据的最值。

可以用 C++ 自带的 priority_queue 维护，也可以自己手写二叉堆。

手写二叉堆

二叉堆要维护一个区间的最值，要支持两个操作：上浮和下沉。

上浮能使这个数据结构里的最小值 “浮” 到堆顶，下沉能使这个数据结构里的较大值 “沉” 到堆中或者堆底。

因为他是二叉堆，所以非常好维护，但是他的一个插入（上浮）或一个删除（下沉）操作就要用 \(O(\log n)\) 的时间复杂度。

这里的堆的编号与线段树相似，左儿子是 \(2x\)，右儿子是 \(2x+1\)。但是这里特殊的一点就是不能有空结点。

插入/上浮

首先把一个数据插入到堆的 \(len\) 的后一个节点，这个点可能与编号为 \(len\) 的点在一层，也可能这个点在它的下一层。之后 \(len \leftarrow len + 1\)。以小根堆为例，为了维护堆顶的元素必须是整个堆的最小值，插入后，若他的父亲节点数值比这个点大，那么就交换该点和父亲节点，然后指针指向父亲节点。如此往复，直到他在堆顶或是它的值比它父亲的大。

削除堆顶/下沉

首先把堆顶删了，然后再把编号为 \(len\) 的点拿出来，放到堆顶。这时发现堆顶的数可能不是最小的，所以要下沉，把下面比这个数小的移上来。操作与上浮类似，把这个点与他的儿子比较，找到他数值最小的儿子，交换它与它的数值最小的儿子节点，指针指向数值最小的儿子节点。如此往复，直到他在堆底或是它的值比它儿子的小。

例题

P1090 合并果子

这道题是一个经典贪心，维护一个小根堆，把他最小的两个数取出来，然后合并这两个点（ans += top1 + top2），再把这个相加的结果放进堆里。因为这个堆是实时更新的，所以可以保证取出来的两个值绝对是堆中最小的两个值。模拟即可。

P1168 中位数

首先解决第一项。1个数的数列他的中位数就是它唯一的元素。我们假设这个数为 \(mid\)。然后再解决第 \(2\sim 3\)项。我们钦定第一项为 \(mid\)，设置一个大根堆和一个小根堆。将这两项中大于等于 \(mid\) 的数插入小根堆，小于 \(mid\) 的数插入大根堆。容易发现这两个堆的元素个数很有可能不等，这时 \(mid\) 就不是中位数了。

再考虑把元素多的堆的元素向 \(mid\) 转移，\(mid\) 再向元素少的堆转移，就能使两个堆的元素个数一样多，\(mid\) 就是中位数了。后面的操作同理。

P2085 最小函数值

~啊→啊→啊↓咦↗啊↓咦↗啊→啊↑啊↓啊～嗯↓哎↓哎↓哎↑哦↑哎→嗯↓哦→哎↑爱爱爱爱爱~~

因为这道题特殊的范围： \({x\in\mathbb{N}*}\)，又因为 \(A, B, C \ge 1\)，可以发现这个函数的最小值点的 \(x\) 值是 \(-\frac{B}{2A} \lt 0\)，所以当 \({x\in\mathbb{N}*}\) 时，\(y\) 一定随 \(x\) 的增大而增大。所以这个函数的最小值一定从 \(x = 1\) 开始， \(1\) 被选了然后选 \(2, 3\)...

再看这道题有很多个函数，容易发现这些函数都是 \(x = 1\) 时 \(y\) 最小。于是我们定义一个结构体，里面有函数的 \(y\) 值和这个 \(y\) 对应的 \(x\) 值。再以 \(y\) 为关键字建一个小根堆，把所有 \(x=1\) 的情况全部 push 进去。取最小值之后，为了不遗漏情况，还要把这个最小值对应函数的 \(x\) 值 \(+1\) 的情况也给 push 进去，然后简单维护即可。

P2827 蚯蚓

虽然这道题似乎和堆完全没有关系......但是85分是可以用堆硬搞出来的。

85pts

模拟，建立一个大根堆，最开始把所有蚯蚓的长度全部都 push 到一个大根堆里。每次要修改的时候，把长度最长的一直蚯蚓取出来，然后把他分成两段，再 push 回大根堆里。但是把其他元素全部加 \(q\) 是不现实的，所以我们可以定义一个偏移量 \(\Delta\)，每次取出最大值的时候要加上这个 \(\Delta\)，改了之后再减去 \(\Delta + q\)，再把偏移量加上 \(q\) 即可。但是注意到 \(m = 7 \times 10^6\)，这个的时间复杂度是 \(O(m\log (n+m))\)，所以会TLE。

100pts

和堆没啥关系。

但和队列有关系。

我们可以考虑沿用85pts的思路，取出里面的最大值然后切除，再把它塞到最小值。

（但是如何维护？最开始的那 \(n\) 个元素还好，后面切除了之后又怎么让切除的两段走到对的位置上去？）

我们知道最开始那n个元素排序了之后是单调不增的，那么把这些元素分别切除（先不增加 \(q\)）的那些长度会不会也是单调不增的呢？

容易发现 \(\lfloor px_1\rfloor \ge \lfloor px_2\rfloor\)，那另外一段呢？

考虑证明：已知 \(x_1 \ge x_2, 0 < p < 1\)，如何证明 \(x_1 - \lfloor{px_1}\rfloor \ge x_2 - \lfloor px_2 \rfloor\)？

首先根据条件可得： \[ x_1 - x_2 \ge p(x_1 - x_2) \] 移项： \[ x_1 - x_2 + px_2 \ge px_1 \] 然后都向下取整\(^{\dagger}\)： \[ \lfloor x_1 - x_2 + px_2 \rfloor \ge \lfloor px_1\rfloor \] 之后把 \(x_1\) 和 \(x_2\) 提取出来，因为是整数，所以结果不变： \[ x_1 - x_2 + \lfloor px_2\rfloor \ge \lfloor px_1\rfloor \] 然后移项即得证。

（\(^{\dagger}\)：已知 \(a \ge b\)，如何证 \(\lfloor a\rfloor \ge \lfloor b \rfloor\)？首先把 \(a, b\) 分为整数部分和小数部分。若整数部分相同，那么忽略小数部分，可得 \(\lfloor a \rfloor = \lfloor b \rfloor\)。若整数部分不同，那么\(\lfloor a \rfloor > \lfloor b \rfloor\)。）

我们维护3个队列，第一个队列从大到小存放开始的那些长度，第二个队列从大到小存放 \(\lfloor px\rfloor\)，第三个队列从大到小存放 \(x - \lfloor px \rfloor\)。因为之前证明了这个单调性（第一个单调，那第二个和第三个也单调），所以就可以在3个队列里选个最大值，把最大值 pop了，再把最大值加上 \(\Delta\) 分为两段，然后把切出来的两段处理后分别 push进第二个和第三个队列。这样就舍去了这个 \(\log\)，复杂度线性。

P3045 Cow Coupons G

反悔贪心 + 堆。

首先我们先选取 \(C\) 值最小的 \(K\) 头奶牛，这是显然的，因为这样可以使前 \(K\) 个奶牛总花费最小。然后后面的怎么贪心？如果后面的选到了一个更好的，我们可以反悔。把之前选了的一头奶牛不用 \(C\) 购买，用原价购买，然后这头牛再用 \(C\) 购买，只能这头购买了比之前的某一头更优才能选这头。比之前更优的条件是什么？

假设现在一共花费了 \(tot\)，之前选了的有一头的编号为 \(j\)，当前看上的这一头编号为 \(i\)。放弃 \(j\) 而选 \(i\) 的条件只有： \[ tot - C_j + P_j + C_i < tot + P_i \] 我们发现 \(tot\)可以删除，再移项可得： \[ P_j - C_j < P_i - C_i \] 设第 \(i\) 头奶牛的优惠力度（\(P_i - C_i\)）为 \(\Delta_i\)，那么要满足： \[ \Delta_j < \Delta_i \] 那么我们可以分别维护以 \(\Delta, C, P\) 为关键字的小根堆，把 \(P, C\) 从小到大选。若选到买过的那就跳过。如果扫到当前的 \(P\) 和 \(C\) 的 \(\Delta\) 大于已选 \(\Delta\) 的堆顶，那么就用优惠券。放弃以前的优惠，选择现在的优惠。每选一个就让当前剩下的钱 \(m\) 减去当前价格，若是反悔那就反悔之前选的再减去现在的优惠价格。当 \(m < 0\) 时，不符合条件，跳出循环。为了方便让前 \(K\) 个元素全用优惠券，就让 \(\Delta\) 堆里面 push \(K\) 个 \(0\)。

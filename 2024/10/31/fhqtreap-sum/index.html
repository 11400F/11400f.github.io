FHQ/无旋平衡树

 

这个平衡树似乎是对新手最友好的平衡树（至少比Splay友好）但似乎又不友好

众所周知，平衡树是一个可以高效进行查找操作的二叉树。一个以值域为关键字的平衡树的节点左边子树的值肯定都比这个点小，右边子树的值肯定都比这个点大。平衡树可以实现插入、删除、查找排名、前驱、后继等等功能。而 FHQ Treap 则只用到分裂和合并两个操作。

分裂

基本函数：void split(int x, int bas, int &l, int &r)。这个函数执行把以 \(x\) 为根的树按照值 \(bas\) 进行分裂，小于等于 \(bas\) 的放到左树，大于 \(bas\) 的放到右树。并且左树和右树也是两棵平衡树。\(l\) 和 \(r\) 就分别返回了左树和右树的根。

合并

个人觉得合并理解起来比分裂简单的很多。基本函数：int merge(int l, int r)，合并以 \(l\) 和 \(r\) 为根的两棵平衡树，再返回这一棵大平衡树的根。这里依据的则是随机分配的优先级。也就是怎么合并完全是随机的。这样才能使复杂度均摊到 $ O (n)$。合并的隐含条件是以 \(l\) 为根的这颗平衡树的所有节点的值都要小于以 \(r\) 为根的所有节点的值。

只要理解了这两个操作，其他的操作都是简单的。

其他操作：

  • 插入值 \(x\)：将树以值 \(x\) 分裂为两棵树，再将 \(x\) 插入左树，再把左树和右树合并。
  • 删除值 \(x\)：分两种情况：
    • 若有多个则只删除一个 \(x\)，其他 \(x\) 保留：分裂出小于 \(x\)，等于 \(x\)，大于 \(x\) 这三棵平衡树，将等于 \(x\) 的这一棵的左子树和右子树合并，这样就消掉了他的根，大小刚刚好 \(-1\)，再分别合并这三棵树。
    • 若有多个 \(x\) 则全删除：分裂出小于 \(x\)，等于 \(x\)，大于 \(x\) 这三棵树，将小于 \(x\) 和大于 \(x\) 这两棵树合并。这样等于 \(x\) 的这一棵树（可能有多个元素）就被完全消掉了！
  • 求数 \(x\) 的排名（小于 \(x\) 的数的个数 \(+1\)）：这个要记录以每个节点为根的子树的大小。定义这个变量为 siz。把他塞到每个节点的结构体里。每次更新的时候再更新一下当前节点的 siz 大小：tr[x].siz = tr[tr[x].ls].siz + tr[tr[x].rs].siz + 1。查找排名即分裂出小于等于 \(x-1\)（即小于 \(x\)）和大于等于 \(x\) 的两棵树，然后排名就是小于 \(x\) 的那棵树的 $ siz+1$。
  • 求第 \(k\) 大数：看这个根的左子树的大小，记其为 \(lftsiz\)。若 \(lftsiz +1= k\)，那就说明第 \(k\) 大数就是根，就返回根。若 \(lftsiz + 1 > k\)，说明第 \(k\) 大数在左子树里，指针指向左子树的根，继续求第 \(k\) 大数。若 \(lftsiz + 1 < k\)，说明第 \(k\) 大数在右子树里，指针指向右子树的根，因为这个右子树没有记这个左子树和这个根，并且根据平衡树的性质，右子树的每一个数一定比左子树的每一个数和根大。那么就查找右子树的排名第 \(k - lftsiz - 1\) 的数即可。递归求即可。
  • 求数 \(x\) 的前驱（第一个比 \(x\) 小的数）：分裂为小于等于 \(x-1\)（小于 \(x\)）和大于等于 \(x\) 的两棵树，记小于等于 \(x-1\) 的树的大小为 \(s\)，那就找这棵树的第 \(s\) 大数即可。
  • 求数 \(x\) 的后继（第一个比 \(x\) 大的数）：分裂为小于等于 \(x\) 的和大于 \(x\) 的两棵树，求大于 \(x\) 的这棵树的第 \(1\) 大（排名为 \(1\)）数即可。

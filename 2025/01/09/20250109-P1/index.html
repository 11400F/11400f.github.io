P11490 Staring Contest 题解

11490DX Lv4

被题目编号吸引进来的一位。

首先我们可以考虑 \(3\) 个整数：假设其为 \(a_1, a_2, a_3\)。这三个整数因为在题目中是两两不同的，所以根据它们的相对大小关系，赋值为 \(0, 1, 2\)，共有 \(6\) 种情况，如下图：

可以发现，我们可以通过查询这 \(3\) 个数的两两之间的 \(\min\) 值，就可以找到那个最小值的位置和最小值的值。

算法 1：\(q = 3(n-2)\) 做法

所以可以根据这个来从左到右每 \(3\) 个数暴力枚举，找到最小值就把它给踢掉，然后再新插入一个数作为 \(a_3\) 继续暴力。每次暴力询问 \(3\) 次，一共就是约 \(3n\) 次，明显过大，通过不了此题。

算法 2：\(q = 2n-3\) 做法

我们发现询问完了把一个数踢掉了之后，剩下的那两个数就会平移到前面作为 \(a_1, a_2\)，而这两个数的 \(\min\) 值是之前问过的，就可以将其代入而无需询问。这样就可以做到新插进来一个数而只需询问 \(2\) 次的更优的算法。但是通过不了 \(q = n+25\) 的点，算法仍需优化。

以下定义前缀次大值个数为 \(s\)。

算法 3：\(q = n + s\) 做法

我们再看上面的那个表，其实问 \(3\) 次都多余了。把表的最后一项 \(\min(a_1, a_3)\) 项删除，会发现并没有什么影响。因为关系就那么几种，所以只需要问两次，即 \(\min(a_1, a_2)\) 和 \(\min(a_2, a_3)\) 的值即可得到最小值的位置和数值。所以当新的一个数插进来时，只需问 \(1\) 次就可以求得最小值和最小值的位置。当然这对最小值在 \(a_1\) 和 \(a_3\) 时是对的，而当最小值在 \(a_2\) 时情况比较特殊。因为下一次询问还需要当前 \(\min(a_1, a_3)\) 的前提。

分析这种情况：根据之前提到的，每次操作都是把最小值剔除然后向前平移到 \(a_1, a_2\)，我们可以得到新 \(a_1\) 和 \(a_2\) 一定是新 \(a_3\) 加进来之前的前缀最大值和前缀次大值。当然顺序还不知道。于是当 \(a_2\) 作为最小值的时候，要么就是前缀最大值更新了，要么就是前缀次大值更新了。我们发现前缀次大值更新是完全包含前缀最大值更新这种情况的。因为当前缀最大值更新时，旧的前缀最大值就会作为新的前缀次大值。所以就可以将其当作前缀次大值更新。综上，当 \(a_2\) 作为最小值时，一定是前缀次大值更新了，由于在这种情况中询问次数要加上 \(1\)，所以就相当于总询问次数加上了总的前缀次大值个数。

算法 4：期望询问次数 \(q = n + 2H_n\)

注：这里的 \(H_n\) 为调和级数，即 \(\displaystyle\sum_{i=1}^{n} \frac{1}{i}\)。

因为如果使用算法 \(3\)，某些良心（？）的出题人就会卡你 corner，直接造一个前缀次大值个数极大的 corner case，就会通过不了。那么我们只能把序列 random_shuffle 一下，竟然通过了此题！这里证明一下询问次数为什么在 random_shuffle 之后就是对的。

因为序列重新打乱过了，所以这里的复杂度就会是期望询问次数。而打乱过后的期望询问次数依然是 \((n + s)\)。之前说到前缀次大值个数可以通过两种情况更新：

  • 新的前缀最大值更新了旧的前缀最大值，而旧的前缀最大值就成为了新的前缀次大值。

    这种情况就要求期望前缀最大值的个数。我们首先可以把序列离散化一下，就变成了一个 \(1 \sim n\) 的排列。

    容易如果数 \(x\) 成为了前缀最大值，那么比 \(x\) 大的数就一定在 \(x\) 的后面，所以留给前面的空间至少也必须要有 \(n-x+1\)。

    这种情况下 \(x\) 对答案的贡献就是 \(1 \times \frac{1}{n-x+1} = \frac{1}{n-x+1}\)。所以期望前缀最大值的个数就是 \(\displaystyle\sum_{i=1}^{n} \frac{1}{i} = H_n\)。

    注：\(H_{1500} ≈ 7.89\)。

  • 新的数比前缀最大值小，但比前缀次大值大。这种情况也可以用类似上面的证明求，这种数的期望也不会超过 \(H_n\)。

所以这种期望询问次数 \(q = n + 2H_n < n + 25\)，可以通过此题。

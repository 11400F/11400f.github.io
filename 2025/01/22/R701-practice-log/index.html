[ABC339E] Smooth Subsequence

dp 式子是可以看出来的。设 \(dp_{i}\) 为在以位置 \(i\) 结尾的最长光滑序列的长度。

于是式子可以列出来： \[ dp_i = \max_{j<i \land a_j \in [a_i-D, a_i + D]}(dp_j + 1) \] 直接用这个鬼玩意的复杂度是 \(O(n^2)\) 的，过不了此题。

于是开始优化：这里使用线段树优化。创建一个值域线段树。对于值域线段树的位置 \(a_x\)，存储的是 \(dp_x\)。于是我们每次只用查找在位置 \([a_i - D, a_i +D]\) 里面的最大值，然后加上 \(1\) 就是 \(dp_i\) 了。最后只需要求 \(dp\) 序列的最大值。

时间复杂度 \(O(n\log n)\)。

Double Exp: Flat Subsequence

[ABC353G] Merchant Takahashi

Tips：最开始高桥的初始位置为村庄 \(1\)。题目翻译忽略了这一点。

设 \(dp_i\) 为赶了第 \(i\) 次集所持有的钱数。因为钱数可以为负数，所以最开始先 memset dp 序列为负无穷，\(dp_0 = 0, t_0 = 0\)。

然后得出 dp 转移式子： \[ dp_i = \max_{j < i} (dp_j - |t_i - t_j|)) + p_i \] 直接用这个鬼玩意的复杂度是 \(O(n^2)\) 的，过不了此题。

于是开始优化：这个绝对值是非常烦的鬼玩意，于是我们将绝对值拆开： \[ dp_i =p_i + \max \left\{ \begin{aligned} dp_j - t_i + t_j) &(t_j \le t_i)\\ dp_j - t_j + t_i) &(t_j > t_i) \end{aligned} \right. \] 于是我们可以将其转移到线段树上。节点 \(t_x\) 存储两个东西： \[ pre_{t_x} = dp_x + t_x\\ suf_{t_x} = dp_x - t_x \] 然后每次查询的时候就查 \([0, t_i]\) 的 \(pre\) 的最大值减去 \(t_i\)，再查 \((t_{i}, MAXN]\) 的\(suf\) 的最大值加上 \(t_i\)，然后两者取 \(\max\)，最后加上 \(p_i\) 就得到 \(dp_i\) 了。

最后答案还是整个 \(dp\) 序列的 \(\max\)。时间复杂度 \(O(n\log n)\)。

Linear Kingdom Races

这道题是一种新的设 DP 状态的一道基础题。

跟 THUWC 2025 D1T1 比较像（给我气笑了(╬▔皿▔)╯）。

还是先设 dp 状态。设 \(dp_i\) 为修复了前 \(i\) 条路的一些路，所得到的最大收益。

于是可以得出下列转移式子： \[ dp_i = \max(\max(dp_j + 在 [j+1, i] 的区间收益), dp_{i-1}) \] 第一项就是修 \(j \sim i\) 的路所带来的钱值的变化，第二项就是啥都不修，光吉猛修。

直接用这个鬼玩意的复杂度是 \(O(n^2)\) 的，过不了此题。

于是考虑优化：还是把它搬到线段树上。每一个线段树节点 \(x\) 存储 \(dp_j + 在[j+1, i] 的区间收益\)，所以这个东西是随着 \(i\) 实时更新的。当 \(i\) 为某个区间 \(x = [l,i]\) 的终点的时候，就把这颗线段树的 \([1,l-1]\) 全部加上 \(p_x\)。

然后就和平常的线段树优化一样搞就可以了。时间复杂度 \(O(n\log n)\)。

P9871 [NOIP2023] 天天爱打卡

这道题是上一道题的升级版。就是加了一个 \(k\) 的限制条件和范围为 \(1e9\) 的 \(n\)。后者可以用离散化解决，前者就用双指针 \(x, i\) 来维护。这里的 \(x\) 表示连续跑步范围边界。当 \(dis_i - dis_x + 1 > k\) 时，\(x\) 自加，知道连续跑步的时间小于等于 \(k\) 天。

Intervals

设 \(dp_{i,j}\) 为考虑区间范围 \([1, i]\)，最后一个 \(1\) 放置于 \(j\) 处的答案。

于是可以得出： \[ dp_{i ,j} = \max_{x \in [1,j]}(dp_{j,x}) + 在 [1,i] 区间内包含点 j 的区间所带来的贡献 \] 直接用这个鬼玩意的复杂度是 \(O(n^2)\) 的，过不了此题。

于是考虑优化：建一个线段树，线段树的每一个节点 \(x\) 表示考虑区间 \([1,i]\) 时，的 \(\max_{y \in[1,x]}(dp_{x,y}) + 在[1,i]区间内包含点 j的区间所带来的贡献\)。这棵线段树是随着 \(i\) 的改变而改变的。每一次遇到以 \(i\) 为终点的区间时，将线段树区间 \([u,i]\) 加上该区间的值。然后第一维 \(i\) 的最大值就是 \(tr_1\)。

时间复杂度 \(O(n\log n)\)。

[AGC044E] Random Pawn

学长讲题一则。

题目描述的不是很清楚，我在这里重新描述一下：

有一个长为 \(n\) 的环，环上的每一个位置上面有权值 \(a_p\) 和代价 \(b_p\)。一开始你会在这 \(n\) 个位置随机一个位置作为起始点。每次你到一个点 \(x\) 的时候，你有两种选择：

  • 继续走其他点，你会随机到达这个位置的相邻的两个位置的其中一个。你会花费 \(b_x\) 的代价。

  • 在该点停止，并且获得该点的权值 \(a_x\)，游戏结束。

求最大的期望价值。

首先我们设在点 \(x\) 获得的最大期望价值为 \(f_x\)。

一个环上是不好搞的，于是我们要先拆环为链。

又因为，在这道题很容易发现的、如果一个位置它的值是这个序列最大的，那么我们也就没必要跳到其他点去得到更小的值。

于是我们就设这条链的两端就为这个最大值，再以这个最大值为开头重新排列。于是把长为 \(n\) 的环拆为了长为 \(n+1\) 的一条链。（同时 \(a, b\) 序列也重新排列。）

为了更好的描述，下面把 \(n \gets n+1\)。

于是就可以得到： \[ \left\{ \begin{aligned} &f_0 = a_0, f_n = a_n\\ &f_i = \max(a_i, \frac{f_{i-1} + f_{i+1}}{2} - b_i) (i \neq 0 \land i \neq n) \end{aligned} \right. \] 我们发现后面的那一坨 $ -b_i$ 很烦，我们现在要把它给消掉。

设 \(g_i = f_i + d_i, c_i = a_i + d_i\)。

于是原式就变为了： \[ \left\{ \begin{aligned} g_0 &= c_0, g_n = c_n\\ g_i &= \max(c_i, \frac{f_{i-1} + f_{i+1}}{2} - b_i + d_i) (i \neq 0 \land i \neq n) \\ &= \max(c_i, \frac{g_{i-1} + g_{i+1}}{2} + \frac{d_{i-1} + d_{i+1}}{2} - b_i + d_i) \end{aligned} \right. \]

然后为了消掉这个烦人的 \(b_i\)，就要使 \(\frac{d_{i-1} + d_{i+1}}{2} - b_i + d_i= 0\)。那么它的递推式就是： \[ d_{i+1} = 2(b_i - d_i) - d_{i-1} \] 于是我们随便设 \(d_0\) 和 \(d_1\)，然后后面的 \(d\) 就可以通过这两个 \(d\) 和整个 \(b\) 序列推出来。

于是就变为了： \[ \left\{ \begin{aligned} g_0 &= c_0, g_n = c_n\\ g_i &= \max(c_i, \frac{g_{i-1} + g_{i+1}}{2}) (i \neq 0 \land i \neq n) \end{aligned} \right. \] 发现 \(g_i\) 的转移式后面的那个玩意像中点公式，于是就可以把它转移到二维平面上：

每一个 \(g\) 在二维平面上就转移成了点 \((i, g_i)\)。如果不考虑 \(c_i\)，那么每一个点就是它的左右两个点的中点，整个 \(g\) 序列在二维平面上会变为一条线段。

现在，\(c_i\) 来了。如果这个时候的 \(c_i\) 大于这个时候的在线段上的值，那么就将这个 \(x\) 值在线段上的点拔高至 \(c_i\)。

现在，第二个点 \((2,4)\) 来了。它因为大于这个时候的线段最大值，所以就把 \(x=2\) 的值拔高到 \(4\)，同时修改 \(g_1\)：

然后第三个点 \((3,3)\)，因为它不大于这个时候 \(l_{CZ}\) 的值，所以它的值是 \(l_{CZ}(3)\)：

第四个点 \((4,5)\)。很明显它大于这个时候的 \(l_{CZ}\)。所以拔高，\(g_4 = 5\)，同时修改 \(g_3\)：

最后，我们发现，这玩意其实就是一个凸包。我们就求凸包推出 \(g\) 序列，再从 \(g\) 反推回 \(f\)。最后因为是随机的一个起点，所以最后答案是整个 \(f\) 序列的平均值。

[ABC389E] Square Price

ABC 389。这场我打了 1147 名。恶臭，E 题比 F 题难。

因为购买 \(k\) 个第 \(i\) 种商品开销 \(k^2P_i\) 元，所以可以看作每买第 \(k\) 个第 \(i\) 种商品开销 \((k^2-(k-1)^2)P_i = (2k-1)P_i\) 元。

那么我们就可以贪心选了。在这 \(n\) 种商品里面选开销最小的选，然后一直选到不能选为止。

但是暴力做是可以构造一组价格全是 \(1\)，\(m =10^{18}\) 的数据来卡你的。所以要优化。

我们枚举我们能买到的最大价格 \(x\)。就相当于是，价格为 \(x\) 及以下的商品全买。价格为 \(x+1\) 及以上的不能全买（但是可以买一些）。我们用二分搜索可以 \(O(n \log n)\) 查找出来这个 \(x\)。然后再 \(O(n)\) 遍历每一个商品，看看有没有并且能不能买价格为 \(x+1\) 的商品。如果有就买。

于是这道题就做完了。

[ABC389F] Rated Range

原神题。

有了数据结构，就不需要脑子了！

因为值域是在 \(5\times 10^5\)，于是我们就可以开一棵线段树，每个线段树的节点 \(x\) 表示的就是初始 Rating 为 \(x\) 时，经过这些 Contest 之后它的最终 Rating 是多少。

因为不管举行多少次，不管加分的 Rating 范围是多少，最终的序列一定是单调的。于是我们就可以线段树二分找到要修改的最小下标和最大下标。然后区间加。最后查询的时候就查线段树上的叶子节点就行了。

所以这道题其实是线段树上二分板子题。然后就做完了。

P4159 [SCOI2009] 迷路

这道题如果没有边权，那么这道题就是邻接矩阵快速幂板子。但是现在有边权。

于是因为边长在 \([1,9]\) 之间，所以可以把一个点拆成 \(9\) 个点。我们设这九个点为 \(a_{x,1}, a_{x,2}, \cdots, a_{x,9}\)。

然后初始时 \(a_{x,i+1}\) 向 \(a_{x,i}\) 连边。

当我们要连边 \((u,v,w)\) 时，就可以从 \(a_{u, 1}\) 连到 \(a_{v,w}\)。因为 \(a_{v,w}\) 向 \(a_{v,1}\) 距离为 \(w-1\)，再加上 \(a_{u,1}\) 到 \(a_{v,w}\) 的边，那么距离刚好就是 \(w\)。然后就可以用邻接矩阵快速幂做了。

快速幂复杂度 \(O((9n)^{2}\log t)\)。

Power of Matrix

这道题让我们求 \(A + A^2 + A^3 + \cdots + A^k\)。\(k\) 太大，那么就用分治。

  • 当 \(k\) 为偶数的时候，那么设 \(mid = \frac{k}{2}\)，那么原式就为 \((A+A^2+\cdots+A^{mid}) + A^{mid} \times (A + A^2 + \cdots + A^{mid})\)。

  • 当 \(k\) 为奇数的时候，那么设 \(mid = \frac{k+1}{2}\)，原式就为 \((A+A^2+\cdots +A^{mid-1}) + A^{mid} + A^{mid+1}(A+A^2+\cdots +A^{mid-1})\)。

然后就可以分治做了。

The Battle of Chibi

这道题还是 dp。

设 \(dp_{i, j}\) 为以 \(i\) 结尾的，长度为 \(j\) 的严格上升子序列的个数，那么就有转移式子为： \[ dp_{i, j} = \sum_{k < i \land a_k < a_i} dp_{k, j-1} \] 初始状态设 \(dp_{i, 1} = 1\)。

直接用这个鬼玩意的复杂度是 \(O(n^2m)\) 的，过不了此题。

于是开始优化：可以用树状数组优化。因为求的是前缀和。但是值域太大，需要先离散化。

时间复杂度 \(O(nm\log n)\)。

P2605 [ZJOI2010] 基站选址

这道题依然是一道 dp。

开始设状态：设 \(dp_{i, j}\) 为将第 \(j\) 个通讯基站设在 \(i\) 位置、并且只考虑区间 \([1,i]\)时，需要花费的最小钱数。那么就可以开始转移： \[ dp_{i, j} = c_i + \left\{ \begin{aligned} &\text{将基站设置在位置 i 时前面没照顾到的人要赔的钱} &(j = 1)\\ &\min_{l < i}(dp_{l, j-1} + \text{将基站设在位置 k, i 时中间没照顾到的人要赔的钱}) &(j > 1) \end{aligned} \right. \] 直接用这个鬼玩意的复杂度是 \(O(n^2k)\) 的，过不了此题。

于是考虑优化：我们首先求出对于村庄 \(i\)，这个村庄能允许的修基站的范围 \([st_i, ed_i]\)，超过这个范围，你就要赔钱。

然后当 \(i > 1\) 时，因为这个是个带范围的 \(\min\) 式，所以可以将其搬到线段树上去做。

具体地、首先将上一个 \(j\) 的 dp 状态搬到线段树上，即节点 \(x\) 设初始值为 \(dp_{x,j-1}\)。然后每次算完 \(dp_{i,j}\) 时，就可以看哪些范围 \([st, ed]\) 的 \(ed\) 值为 \(i\)。若存在，就在节点 \([1,st-1]\) 区间加上该区间对应的 \(w\)。然后算 \(dp_{i,j}\) 就直接查区间 \(\min\) 即可。

而当 \(i=1\) 时，方法就不一样，但思路是类似的。因为这是建的第一个基站，所以就不存在什么 \(dp_{l, j-1}\) 一说。而赔的钱，自然也就是把基站建在 \(i\) 的前面所有在范围外的村庄的赔的钱的总和了。

P3287 [SCOI2014] 方伯伯的玉米田

这道题也是一道 dp。用人话来说，就是可以执行最多 \(k\) 次区间加 \(1\)。然后求最长单调不下降序列。

我们还是先设 dp 状态：设 \(dp_{i, j}\) 为以 \(i\) 结尾，并且 \(i\) 被拔高了 \(j\) 次所构成的最长单调不下降序列。

我们能发现一个性质：就是你要拔高一个区间，那么拔 \([i, n]\) 肯定是最优的。因为如果你不是以 \(n\) 结尾，那么中间的数和右边的数的相对大小关系就会改变，就会使右边的数相对中间的数变小。这样肯定是不优的，因为我们要求最长单调不下降序列。

然后就可以得出 dp 转移式： \[ dp_{i, j} = \max_{p < i, q \le j, a_{p} + q \le a_i+j}(dp_{p, q})+1 \] （不用考虑是不是不连续了，因为反正中间的那些你都不管，那么根据之前的性质，就让它加上 \(j\)。）

但是，直接用这个鬼玩意的复杂度是 \(O(n^2k)\) 的，过不了此题。

于是考虑优化：我们可以看到转移式是个 \(\max\) 式，限制条件有 \(3\) 个（其实从左到右枚举可以消掉第一个变成 \(2\) 个）。然后如果只有 \(1\) 个限制条件就可以把它放到树状数组上做，那现在 \(2\) 个限制条件就可以把它放到 \(2\) 维树状数组上做。

于是这道题就做出来了，时间复杂度 \(O(nk\log n\log k)\)。

Codeforces 2063-D

这道题赛时想到了思路，但是因为时间不够没有调出来。然后掉到了 1800 多名去了（Codeforces Round 1000）。Yee tour fen.

容易发现，选线段肯定是选最两边的，选点肯定是选最中间的。这样是最优的。那么我们首先给点排序（题目里没说序列有序！！），然后就可以开始贪心选：每次选线段就比较一下 \(a\) 侧的两端的距离和 \(b\) 侧两端的距离。然后选最长的那一端，假设为 \(a\) 端。然后把两端的两个点和 \(b\) 端的中间的那一个点给删掉。然后把这个三角形记录到 \(a\) 栈里面（栈的作用后面再说）。如果在 \(b\) 端同理。

但是这个时候，你会遇到在 \(a\) 端选完了，但是 \(b\) 端还剩一堆点的情况。这个时候因为还没有达到 \(k_{max}\)，所以肯定是要强制选对面的一些点的。那么我们就从 \(a\) 栈里面弹出来一个三角形（就是删除一个三角形。根据从大到小选的原则，这个三角形是面积最小的）。这样就给对面的点创造出来了 \(2\) 个 \(a\) 点。然后对面的 \(4\) 个 \(b\) 点再和这 \(2\) 个 \(a\) 点相组合，就是 \(2\) 个新的三角形。再减去弹出来的那个三角形，就相当于这就是多了一个三角形的最优答案。

那么边界 \(k_{max}\) 如何算？这个其实是好算的。设最终选的最多次在 \(a\) 端选了 \(p\) 条线段，在 \(b\) 端选了 \(q\) 条线段。

那么就可以列出不等式：

  • \(2p + q \le n\)
  • \(2q+p \le m\)

联立可得 \(k_{max} = p+q \le \frac{n+m}{3}\)。

又因为两侧选点的极限是 \(n, m\)，所以 \(k_{max} = \min(\lfloor\frac{n+m}{3}\rfloor, n, m)\)。

于是这道题就做完了。

AT_dp_s Digit Sum

从这里开始似乎就都是 AT_dp 比赛的题了。

这道题一眼数位 dp。

首先设状态：设 \(dp_{pre, now, remain}\) 为在没有到边界的情况下，当前面的数的总和 \(\bmod d = pre\) 时，并且取的上一个数为 \(now\)，并且还剩下 \(remain\) 个数没有取时的答案。然后像一般的数位 DP 记忆化 + 转移即可。

AT_dp_t Permutation

这道题的 DP 状态比较巧妙。

设 \(dp_{i, j}\) 表示现在一共填了 \(i\) 个数，并且刚刚新加进来的数是在这 \(i\) 个数里面从小到大的第 \(j\) 个。

为什么这么设状态呢？因为其实这些大于号小于号是用来限制它们的相对大小关系的。只要相对大小关系对了，怎么填都可以。

于是我们就可以轻松得出式子： \[ dp_{1, 1} = 1 \]

\[ dp_{i, j} = \left\{ \begin{aligned} &\sum_{k=1}^{j-1} dp_{i-1, k} &(s_{i-1} = '<') \\ &\sum_{k=j}^{i-1} dp_{i-1, k} &(s_{i-1} = '>') \end{aligned} \right. \]

直接用这个鬼玩意的复杂度是 \(O(n^3)\) 的，过不了此题。

于是考虑优化：这道题的优化是最简单的。直接前缀和优化即可。时间复杂度 \(O(n^2)\)。

AT_dp_u Grouping

这道题可以想到一个非常简单的 dp：假设 \(val_S\) 代表只考虑 \(S\) 这个集合的最大贡献。于是列式子： \[ val_{S\cup T} = \max_{S \cap T \neq \emptyset}(val_{S\cup T}, val_{S} + val_{T}) \] （初始设 \(val_{S}\) 为把 \(S\) 集合整体分为一组所得的值。）

但是不好枚举，然后转化一下： \[ val_{S} = \max_{T \subseteq S}(val_{S}, val_{T} + val_{\complement_S T}) \] 这样是好枚举的。

直接用这个鬼玩意的理论复杂度是初始化 \(O(n^2 2^n)\) 和 DP 复杂度 \(O(2^{2n})\) 的，但是过得了此题（可能是因为 ATCoder 机子非常的强大）。

但是运行了 1.3s，还是要考虑优化：这个 DP 非常浪费的一个地方就是枚举了非常多的不包含于 \(S\) 的 \(T\)。那么怎么才能枚举所有的 \(T\) 让它们都属于 \(S\) 呢？

有一个方法：因为这里的集合使用一个二进制数表示的，所以可以先从 \(S\) 开始枚举，然后下一个是 (T-1) & S。这种方法可以完美的枚举所有 \(S\) 的子集。

时间复杂度预处理 \(O(n^2 2^n)\)，DP \(O(3^n)\)。

证明枚举子集的复杂度为 \(O(3^n)\) 的博客：枚举子集为什么是 O(3^n) 的 - yspm - 博客园

AT_dp_v Subtree

这道题就是在树上做 dp。（感觉说了和没说一样

因为这是一棵无根树，所以就先钦定节点 1 为根。然后设状态。又因为是在强制把这个节点染成黑色的情况，所以连通要么黑色点连在下面，要么连在上面。而因为遍历一棵树最常用的方法是 DFS，所以我们把 dp 状态分成两个：

  1. \(dpin_x\)：设其为当黑色点全在以 \(x\) 为根的子树内（包含点 \(x\)）的方案数。
  2. \(dpout_x\)：设其为当黑色点全在以 \(x\) 为根的子树外（也包含点 \(x\)）的方案数。

然后总答案就是 \(dpin_x \times dpout_x\) 了。

那我们该如何求呢？首先，对于 \(dpin_x\)，它的答案是根据它的儿子决定的。很容易得出：\(dpin_x = \prod_{v \in son_x} (dpin_v + 1)\)。而这个 \(+1\) 就是当儿子 \(v\) 取白色的情况，其他就是儿子 \(v\) 取黑色的情况。这就相当于是，它有它的儿子的个数种选择，然后求方案数。而当 \(x\) 为叶子节点时，很明显，\(dpin_x = 1\)。

而对于 \(dpout_x\)，因为它要和不是它的子树的点连通，所以肯定就要经过它的父亲。那么就相当于是变为了它的直系兄弟的 \((dpin_x + 1)\) 之积。而又因为 \(dpout\) 是向外面连通的，父亲的祖先也不例外，所以还要多一个选择，即再乘上 \(dpout_{fa_x}\)。最后再加上 \(1\)，表示只选 \(x\) 点，不选 \(fa\) 及其他点。而当 \(x\) 为根时，明显的、\(dpout_x = 1\)。

AT_dp_x Tower

我们可以想到一个朴素的算法：先定一个 \(dp\) 数组，\(dp_i\) 表示目前箱子总重量 \(i\) 所可以获得的最大价值。假如说新加了一个箱子 \(j\)，那么我们可以将 \(dp_0 \sim dp_{s_j}\) 全部更新一遍。（要反序因为是背包）最后求最大值。因为 \(n\le 10^3\)，\(w, s\le 10^4\)，所以时间复杂度就算是 \(O(nw)\) 或 \(O(ns)\) 也可以接受。

但是遇到了一个问题：如何确定放箱子的顺序使最终答案最优？

不容易想到（我看了 tj），现在有两个箱子 \(x, y\)，让 \(x\) 在 \(y\) 前面使这样更优的情况可以做如下解释：现在你假定 \(x\) 和 \(y\) 都要选，那么如何安排顺序使最终答案更优？那就分两种情况讨论：

  • 若 \(x\) 在 \(y\) 上面，那么还可以放 \(s_y - w_x\) 的重量在 \(y\) 上。
  • 若 \(y\) 在 \(x\) 上面，那么还可以放 \(s_x - w_y\) 的重量在 \(x\) 上。

这样就好理解了，哪个的剩余重量大就选哪种情况。所以将 \(x\) 优先于 \(y\) 的情况则当： \[ s_y - w_x \le s_x - w_y \] 于是在开头的 init 部分排序排一下优先级，就做出来了。
